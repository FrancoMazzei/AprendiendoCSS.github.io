Algunos Selectores

Selector de tipo

            i {
                los estilos que querramos aplicar
            }

" i "este es un seleccionador de tipo, y se usa para seleccionar una etiqueta o varias etiquetas de HTML, en este caso, todas las etiquetas i (letra italica), seleccionaria todas las i que haya en el documento.


Selector de id

                <h2 id="titulo-secundario"> Titulo secundario </h2>

En css:

            #titulo-secundario {
                los estilos que querramos aplicar
            }

" # " el hashtag se usa para el id,es para darle un identificador unico, no se puede repitir, solo en un elemento HTML.


Selector de clases.
En HTML:

            <h1 class="titulo"> Titutlo </h1>

        
            .titulo {
                los estilos que querramos aplicar
            }

" . "clases (class) se usa para definnir el estilo de muchos elementos HTML, esto hace que quede todo ordenado y con menos errores de especifidad.

Selector por atributo.

En HTML(por ejemplo):

            <h2 atributopersonalizado="otro-subtitulo">Otro Subtitulo</h2><br>
            <h3 href="https://enlace-imaginario">Enlace Imaginario</h3><br>

En CSS:

            [atributopersonalizado="otro-subtitulo"] {
                los estilos que querramos aplicar
            }
            

            [href="enlacei-maginario"] {
                estilos
            }

" [] "esto se debe usar para los atributos personalizados o no.
Los selectores de atributos tienen bastante potencial, muchas variantes, como:

        .selector-atributos a[href] {
            estilos
        }

Combinandolo con un selector de hijos-descendientes (que esta escrito con clase pero se puede escribir con cualquier selector que se quiera) de atributo. Decimos que solo le aplique los estilos a los elementos que contengan un href en esa clase.

        .selectores-atributos a[target="_blank"] {
            font-size: 22px;
        }

No siempre deben estar solo los atributos, a veces para mayor especificacion se pone el atributo, junto al valor que tenga.

        .selectores-atributos a[href*="espacial"] {
            font-weight: bold;
        }

* se usa para indicar para que solo se le agregue el estilo a los atributos que contengan la palabra puesta, en este caso "especial."

        .selectores-atributos a[href^="http:"] {
            color: orangered;
        }

^ se usa para indicar que solo le debe dar estilo a los atributos que comiencen con "http"

        .selectores-atributos a[href$=".com"] {
            text-decoration: none;
        }

$ al contrario del anterior, este simbolo se usa para especificar que solo se le aplique el estilo a los atributos que terminen con ".com."
Para el que sigue, es con un ejemplo en HTML:

        <li><a class="menu-item animales" href="https://marmotaespacial" target="_blank">Marmota4</a></li>
En css: 
        .selectores-atributos a[class~="animales"] {
            background-color: black;
      }

~ la virguililla se usa para decir que se pongan los estilos a la clase que contiene el texto indicado y que este separado por espacios. Y obviamente no se tendria que poner a href, si no class, por que en HTML class es un atributo.

        .selectores-atributos a[class|="menu"]{
            font-style: italic;
        }

| siguiendo con el ejemplo anterior, este simbolo sirve para decir que los estilos se deben aplicar a la clase que contenga el texto indicado pero que este este despues este separado por un guion medio ( - ).

En este caso usamos un selector class con el selector universal en un elemento HTML para agrandar el tamaño de la letra 20px.

Pseudo Elementos

Los pseudo elementos se usan para dar estilo a parte específicas de un elemento y/o agregar contenido adicional, se representa con dos puntitos ::

        .libro::before {
            content: "📚";
        }

::before se usa para agregar contenido o estilos antes de un elemento o contenido en HTML.

        .libro::after {
            content: "📈";
        }

::after se usa para agregar contenido o estilos despues de un elemento o contenido en HTML.

        .pseudo-elemento::first-letter {
            font-size: 25px;
            font-weight: bold;
        }

::first-letter quiere decir primera letra, le dara estilos a la primera letra de un texto o de lo que se quiera.

        .pseudo-elemento::first-line {
            color: salmon;
        }

::first-line quiere decir primera linea, y le dara estilos a la primera liena de un texto.

        .pseudo-elemento::selection {
            background-color: black;
            color: greenyellow;
        }

::selection sirve para que cuando hacemos copiar y pegar, se marque de un determinado color el estilo al copiar.

        .placeholder::placeholder {
            color: red
        }

::placeholder es para darle estilo a los placeholders.

La cascada

A css se le dice estilos en cascada porque los estilos van cayendo como si fuera una cascada.

Especifidad

La especifidad son las reglas css que determina cuales esilos se aplicaran al elemento HTML.
Con la especifidad podemos aplicar los estilos sin caer en errores, como que si ponemos un estilo a un elemento y luego le ponemos a ese elemento de nuevo otro estilo que no tome este ultimo. La especifidad es muy importante para CSS.

El important mejor no usarlo, pero en casos extremos que se deba usar, esta ahi. Este aplica para todos los selectores CSS.

Reseteo y normalizacion de estilos

Reseteo y normalizacion son maneras de cambiar la configuracion predeterminada del navegador para que este se vea de la misma forma en cualquier navegador, ya que cada navegador tiene una configuracion predeterminada de estilos.
reseteo elimina los estilos predeterminados del navegador, y normalización, reemplaza la configuracion de estilos predeterminada del navegador por otros.
Podemos combinar una hoja de reseteo con una de normalizacion o solo poner una de estas dos (yo diria normalizacion), al final esto dependera de las necesidades y de nuestra comodidad o gustos tambien. Si usamos una hoja de normalizacion bien diseñana y completa, no necesitaremos un reseteo adicional.

Podemos usar la hoja de reseteo, que no la recomiendo porque reinicia TODO y eleimina la configuracion predeterminada del navegador.
O
Usar la hoja de normalización, que es mejor para mi, ya que deja una base solida de configuración para poder trabajar.

La hoja de estilos siempre debe ponerse encima de la hoja css que estemos poniendo los estilos, para que el navegador lea primero a esta y luego a la nuestra.

Modelo de caja

El Modelo de caja y es la forma en que el navegador representa a todas las etiquetas que estan dentro de body. Cada etiqueta genera una caja y el comportamiento de esa caja depende de la etiqueta, si es de linea o de bloque (este comportamiento se puede cambiar con display en CSS). Es muy importante el modelo de caja porque entre todas las cosas para que sirve, en CSS nos ayuda a diseñar bien nuestra pagina porque podemos ver todo lo que ocupan nuestras etiquetas, bordes, altos, anchos, etc.

Hay dos tipos de caja:

Las de bloque (block boxes)

, siempre se ajustarán al tamaño del ancho total que las contiene.


Las de línea (inline boxes)

, solo se ajustarán al tamaño del contenido.

Teoría de unidades

La teoria de medidas es para saber de que forma podemos modificar el tamaño de la mayoria de los elementos HTML.
Hay dos unidades de medidas:
Fijas: son fijas, medidas absolutas que no dependen de una unidad de referencia por ejemplo cm, px, mm.
Relativas: son las que su tamaño depende de un valor de referencia y aca se usan unidades em, rem o %.
em: basa su tamaño en cuestion de una unidad fija.
%: basa su tamaño en cuestion del elemento que lo contiene, como el 16% de un div.
rem: basa su tamaño en cuestion de las medidas dadas en el tamaño de fuente ya definido, que suele ser html 

html {
        font-size: 18px;}

todo lo que contenga rem estaria midiendo lo que dice el tamaño de fuente.

Viewport heigth y width

Viewportse usa para controlar los tamaños de los elementos que se mostraran el la pagina.
viewport heigth(vh) es para controlar el alto o largo y viewport width(vw) es para controlar el ancho.
Ejemplo:

        .htre {
            color: greenyellow;
            font-size: 30px;
            background-color: greenyellow;
            width: 50vw;
            height: 50vh;
        }
        
Toda esta etiqueta que nos envuelve (pre) ocupara 50 de ancho y 30 de alto o largo.

Propiedades de texto CSS

font-size, para modificar el tamaño del texto.
font-family, para cambiar el estilo del texto.
color, modificar el color.
line-heigth, controlar el espacio que ocupa una letra.
font-weigth, modifica el grosor.
text-align, establece donde se vera el texto, si izquierda, derecha, centro, etc.
text-decoration, decora al texto, subrayados, resaltador, etc.
text-transform, cambia el aspecto del texto pero en terminos de mayúsculas o minúsculas.
latter-spacing, define el espacio entre las letras.
word-spaccing, define el espacio entre palabras.
text-shadow, le da sombra al texto.
text-overflow, maneja el comportamiento del texto en caso de que exceda el espacio disponible dentro de su contenedor. Se suele usar con otras propiedades para lograr el efecto deseado, como:
white-space, controla como se maneja los espacios en blanco, su valor por defecto es:
normal, ajusta automaticamente el texto a la anchura del contenedor, respetando los espacios adicionales y saltos de linea.
nowrap, este muestra el texto en una sola linea ignorando espacios adicionales y saltos de linea.
overflow, dice como se comportara el contenido si se exceden los limites. algunos valores:
visible, deja que el contenido pueda salir del contenedor y ponerse encima de otros elementos, desplazándolos hacia arriba o abajo
hidden, elimina a la vista del usuario/a el contenido que exceda el contenedor.
scroll, le pone barras de dezplazamiento incluso si no excede el contenedor.
auto, este solo muestra las barras si de verdad excede el contenedor.
text-overflow, indica como se debe ver visualmente el texto.
clip, recorta el texto excedente, similar a hidden pero solo aplica al texto dentro de un elemento en particular, como div o span.
elipsis, muestra puntos suspensivos al final del texo, diciendo que hay mas contenido.

Position fixeds

El ejemplo lo hice a base del titulo "Aprendiendo CSS" pero con otro porque para que conserve su lugar al subir ya se de usar js.

Aprendiendo css

 
        .fixedtitle {
            background-color: black;
            color: greenyellow;
            top: 0;
            right: 0;
            position: fixed;
         }

Tiene un z-index de 200 para que se ponga por encima de todos los elementos. y fixed va bajando mientras bajamos en la pagina, como si fuera un anuncio o una barra de navegacion. Pero que pasa si el elemento que ponemos nos ocupa espacio osea, queda tapando algo al subir? Se le puede poner al bodo un padding-top de x cantidad de px y al elemento un margin de -x cantidad de px o positivo esos px o a veces ni es necesario aplicarles un margin y con "padding-top" es suficiente, y de esa forma se le asignaria un lugar, es una forma de hacerlo, si no otra manera es como dije, con js.
